# Анализ программы и требований

Дано требование реализовать динамический контейнер на основе динамического массива, соблюдая:

- разделение на интерфейс и реализацию;
- генерацию исключений в конструкторах и методах при необходимости (стандартные исключения);
- арифметические операции (сложение, вычитание, копирование) методами класса;
- операции сравнения (больше, меньше, равно) методами класса;
- неизменяемость объектов — методы возвращают новый объект, исходные не меняются.

## Что реализовано

Класс `Three` представляет неотрицательные числа в системе счисления по основанию 3. Внутреннее представление — динамический массив цифр ('0'..'2') в «перевёрнутом» виде: младший разряд находится по индексу `0`.

- Интерфейс вынесен в `include/Three.h`, реализация — в `src/Three.cpp`.
- Конструкторы: по умолчанию, конструктор размера и заполнителя, из `initializer_list`, из `std::string`, копирования, перемещения. Реализованы копирующее и перемещающее присваивания.
- Валидируются все входные цифры; при ошибке выбрасывается `std::invalid_argument`.
- Методы `Sum` и `Sub` — без побочных эффектов; исходные объекты не изменяются. `Sub` выбрасывает исключение при попытке получить отрицательный результат.
- Операции сравнения `Equal`, `Less`, `More` реализованы корректно (с учётом старших разрядов). `More` определена как `!Equal && !Less`.
- Набор модульных тестов GoogleTest покрывает конструкторы, арифметику, сравнения, неизменяемость и обработку ошибок.

## Ограничения и допущения

- Поддерживаются только неотрицательные значения (без знака). Вычитание, ведущее к отрицательному результату, запрещено и приводит к исключению.
- Ввод цифр вне диапазона '0'..'2' запрещён и приводит к исключению.
